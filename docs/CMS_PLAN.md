# Future CMS requirements

A non-technical editor should eventually manage copy, links, media, SEO, legal details, Speech Lab questions/results, and form labels without editing React code.

Before implementation, compare suitable options such as Decap CMS, Sanity, Directus, Strapi, Payload, or Supabase with a custom admin. Evaluate hosting, authentication, media storage, preview, publishing, version history, rollback, cost, and maintenance.

Do not implement a final admin panel until the user approves the proposed architecture. LocalStorage is not an acceptable production CMS.
