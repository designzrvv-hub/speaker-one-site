# Telegram для Speaker One

В проекте есть две независимые функции:

1. Публичная ссылка открывает Telegram, бота или канал — её можно менять в Sanity.
2. Заявка формы отправляется через защищённую серверную функцию — её секреты настраиваются на хостинге.

## Изменить публичную ссылку на бота

1. Откройте Studio.
2. Выберите **3. КНОПКИ И ССЫЛКИ → Социальные сети и Telegram**.
3. В поле **Telegram-бот** вставьте `https://t.me/имя_бота`.
4. Включите **Показывать**.
5. Нажмите **Открыть и проверить ссылку**.
6. Нажмите **Publish** и пересоберите сайт.

Bot Token сюда вставлять нельзя. Публичная ссылка начинается с `https://t.me/`, а Bot Token выглядит иначе и предоставляет управление ботом.

## Как заявки доходят до Telegram

```text
Форма сайта → серверная функция → Telegram Bot API → чат владельца
```

Браузер никогда не обращается к Bot API напрямую. Сервер возвращает успех только после подтверждения Telegram.

## Что понадобится владельцу

- существующий Telegram-бот;
- Bot Token, полученный у BotFather;
- Chat ID личного чата, группы или канала;
- необязательный Thread ID темы в forum-группе;
- выбранный хостинг серверной функции;
- окончательный домен сайта;
- доступ к secret/environment settings хостинга.

## Как получить Chat ID безопасно

1. Добавьте бота в нужный чат и дайте ему право отправлять сообщения.
2. Отправьте в этот чат тестовое сообщение или команду боту.
3. Получите Chat ID серверным инструментом или временным локальным скриптом разработчика.
4. Не публикуйте полный Chat ID в CMS, клиентском коде или скриншотах.
5. Сохраните значение только в secret settings хостинга.

Для канала Bot ID обычно относится к каналу, а бот должен быть администратором с правом публикации. Для forum-темы дополнительно нужен `TELEGRAM_THREAD_ID`.

## Переменные хостинга

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
TELEGRAM_THREAD_ID=
LEAD_WEBHOOK_SECRET=
LEAD_ALLOWED_ORIGINS=https://ВАШ-ДОМЕН
LEAD_STATUS_ALLOWED_ORIGINS=https://ВАШ-АДРЕС-STUDIO
LEAD_TIMEZONE=Asia/Yekaterinburg
```

На публичном сайте задаётся только URL функции:

```env
VITE_LEAD_ENDPOINT=https://ВАШ-ДОМЕН/api/leads
```

`VITE_LEAD_ENDPOINT` не является секретом. Все остальные значения остаются на сервере.

## Какой хостинг выбрать

Используйте функцию на той же платформе, где будет размещён сайт — это упрощает HTTPS и CORS. В проекте подготовлены адаптеры для Vercel, Netlify и Cloudflare Pages Functions. До выбора платформы ни один адаптер не считается размещённым production endpoint.

## Настройка за 7 шагов

1. Выберите хостинг и подключите подходящий адаптер из `server/adapters/`.
2. Добавьте server environment из предыдущего раздела.
3. Укажите точный production origin без wildcard.
4. Разместите функцию и проверьте её безопасный status endpoint.
5. Укажите URL функции в `VITE_LEAD_ENDPOINT` build-окружения сайта.
6. Пересоберите и разместите сайт.
7. Отправьте тестовую форму и убедитесь, что сообщение появилось в нужном чате.

## Формат сообщения

Telegram получает только данные формы и разрешённые UTM-метки:

```text
Новая заявка с сайта SPEAKER ONE

Имя:
...

Контакт:
...

Что хочет улучшить:
...

Источник:
Сайт SPEAKER ONE

Дата и время:
...
```

IP и секреты в текст сообщения не добавляются.

## Тестовая проверка

Сначала выполните локальные тесты без реального Telegram:

```powershell
npm.cmd run test:lead
```

Затем выполните защищённый server-side тест размещённой функции:

```powershell
$env:LEAD_TEST_ENDPOINT='https://ВАШ-ДОМЕН/api/leads/test'
$env:LEAD_WEBHOOK_SECRET='СЕКРЕТ_ИЗ_ХОСТИНГА'
npm.cmd run lead:test
Remove-Item Env:LEAD_TEST_ENDPOINT
Remove-Item Env:LEAD_WEBHOOK_SECRET
```

После теста отправьте обычную заявку через сайт. Не используйте реальные персональные данные в тестовых сообщениях.

## Если заявка не пришла

1. Убедитесь, что endpoint отвечает и статус в Studio не равен «Не настроено».
2. Проверьте, что бот имеет право писать в выбранный чат.
3. Проверьте Chat ID и необязательный Thread ID.
4. Проверьте exact origin в `LEAD_ALLOWED_ORIGINS`.
5. Проверьте, что `VITE_LEAD_ENDPOINT` вошёл в новую сборку сайта.
6. Посмотрите технические логи функции без вывода Token и полных данных заявки.
7. Оставьте резервную Telegram-ссылку доступной до устранения ошибки.

## Безопасность

- Включите MFA для Sanity и аккаунта хостинга.
- Не отправляйте Bot Token в мессенджерах и не вставляйте его в Sanity.
- При подозрении на утечку немедленно перевыпустите Token через BotFather.
- Ограничьте доступ к secret settings.
- Включите rate limiting платформы: локальный счётчик serverless-функции не защищает от распределённых запросов.
- Ограничьте CORS только реальным доменом и отдельным адресом Studio для health status.

## Что уже готово, а что требует владельца

Готовы обработчик, валидация, timeout, honeypot, exact CORS, безопасный Telegram message, health/test handlers и адаптеры трёх платформ. Для реальной отправки владелец должен выбрать хостинг, добавить секреты, указать домен и выполнить подтверждённый тест в настоящем чате.
