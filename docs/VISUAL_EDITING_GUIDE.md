# Визуальное редактирование Speaker One

В разделе **«Редактор сайта»** сайт открывается прямо внутри Sanity Studio. Текст можно выбрать на странице, а фотографию — нажать, чтобы Studio открыла связанное поле.

## Самый быстрый способ изменить текст

1. Запустите сайт командой `npm.cmd run dev:visual`.
2. Запустите Studio из папки `studio` командой `npm.cmd run dev`.
3. Откройте **«Редактор сайта»**.
4. Нажмите на подсвеченный текст и измените поле справа.
5. Проверьте обычный и мобильный вид.
6. Нажмите **Publish** только после проверки.

## 1. Как запустить сайт

В корне проекта создайте `.env.local` по примеру `.env.example`. Для локального визуального редактора нужны:

```dotenv
VITE_SANITY_PROJECT_ID=project_id_владельца
VITE_SANITY_DATASET=production
VITE_SANITY_API_VERSION=2026-07-15
VITE_SANITY_USE_CDN=true
SANITY_API_READ_TOKEN=локальный_read_token
SANITY_STUDIO_URL=http://localhost:3333
SANITY_PREVIEW_PORT=5173
SANITY_PREVIEW_SESSION_SECRET=
```

`SANITY_API_READ_TOKEN` должен иметь только право чтения. Он не должен начинаться с `VITE_`, попадать в Studio env, Git, screenshot или сообщение. Пустой `SANITY_PREVIEW_SESSION_SECRET` допустим локально: при запуске создаётся временный ключ. Для размещённого preview задайте отдельное случайное значение длиной не меньше 32 символов в server environment.

Запустите защищённый preview-server:

```powershell
npm.cmd run dev:visual
```

Адрес сайта: [http://localhost:5173/](http://localhost:5173/). Обычный `npm.cmd run dev` по-прежнему запускает только Published-версию без черновиков и подсветки.

## 2. Как запустить Studio

В отдельном окне PowerShell:

```powershell
Set-Location -LiteralPath studio
npm.cmd run dev
```

Откройте [http://localhost:3333/](http://localhost:3333/) и войдите аккаунтом владельца.

## 3. Как открыть «Редактор сайта»

В верхнем меню Studio выберите **«Редактор сайта»**. Внутри Studio откроется реальный сайт, рядом — форма документа.

Если вместо сайта показана ошибка, проверьте, что:

- команда `npm.cmd run dev:visual` продолжает работать;
- порт 5173 не занят другим Vite-процессом;
- root `.env.local` содержит Project ID и read-only token;
- dataset называется `production`;
- локальный адрес preview совпадает с `SANITY_STUDIO_PREVIEW_URL`.

## 4. Как нажать на текст

1. Наведите курсор на заголовок, описание, кнопку или текст карточки.
2. Дождитесь рамки редактируемой области.
3. Нажмите на неё.
4. Studio откроет конкретный документ и конкретное поле.
5. Изменение Draft появится на странице без ручной перезагрузки.

Это работает для Hero, навигации, философии, компетенций, блока эксперта, карточек опыта, этапов, Speech Lab, публичных текстов формы и Footer. SEO открывается отдельным документом, потому что его поля не являются видимым блоком страницы.

## 5. Как заменить фотографию

1. Нажмите на фотографию в preview.
2. Откройте выбранное поле изображения.
3. Выберите или создайте документ в разделе **«Медиа»**.
4. Загрузите изображение, заполните понятный alt-текст и при необходимости настройте crop/hotspot.
5. Вернитесь в документ раздела и проверьте preview.
6. Опубликуйте media-документ и документ раздела после проверки.

Поддержаны видимые изображения: логотип навигации, Hero, философия, основной портрет эксперта и фотография Speech Lab. Если на странице показан локальный fallback, он намеренно не подсвечивается как Sanity-контент.

## 6. Как проверить мобильную версию

В панели Presentation Tool переключите размер preview на мобильный. Проверьте прежде всего:

- длинный заголовок Hero и кнопку;
- меню;
- карточки;
- вопросы Speech Lab;
- поля формы;
- Footer и реквизиты.

Перед Publish полезно проверить минимум 320, 375, 768, 1024, 1440 и 1920 px. В текущем редакторе desktop/mobile меняют только окно просмотра и не публикуют данные.

## 7. Draft и Publish

- **Draft** — рабочая версия, видимая в Studio и защищённом preview.
- **Published** — версия, которую может получить следующая публичная сборка.
- Нажатие **Publish** пока не запускает deploy автоматически.
- Настоящий сайт обновится только после `npm.cmd run build` и размещения нового `dist`.

Обычный посетитель никогда не видит Draft: публичная сборка использует только `perspective=published` и статический snapshot.

## 8. Как отменить изменение

- Для неопубликованного Draft используйте **Discard changes**.
- Для опубликованного документа откройте доступную историю и восстановите нужную версию.
- Если CMS-значение отсутствует или не проходит проверку, сайт показывает соответствующий fallback из `siteConfig.js`.
- Не удаляйте singleton-документы и стабильные ключи элементов массива ради отката.

## 9. Если элемент не подсвечивается

Проверьте:

1. сайт открыт именно через **«Редактор сайта»**, а не отдельной обычной вкладкой;
2. запущена команда `dev:visual`, а не `dev`;
3. поле заполнено в Sanity и прошло проверку;
4. массив карточек, этапов или Speech Lab валиден целиком;
5. изображение действительно выбрано из Sanity, а не взято из fallback;
6. после изменения сохранён Draft.

Для нажатия обычных кнопок сайта временно переключитесь из режима выбора полей в режим просмотра в панели редактора. Внешние ссылки открывайте в новой вкладке.

## 10. Если preview не открывается

- убедитесь, что Studio доступна на 3333, а preview — на 5173;
- перезапустите обе команды после изменения `.env.local`;
- проверьте точные origins в Sanity Manage и `SANITY_STUDIO_PREVIEW_URL`;
- убедитесь, что хостинг не запрещает iframe через `X-Frame-Options` или `frame-ancestors`;
- проверьте, что read token не отозван и имеет только право чтения;
- не заменяйте server-side token переменной `VITE_SANITY_TOKEN`.

## 11. Что нельзя вводить в CMS

Нельзя сохранять в поля Sanity:

- read/write token;
- Telegram bot token и chat ID;
- webhook/deploy secret;
- CRM token;
- пароль, recovery-код или приватный API key;
- lead endpoint и серверные credentials.

## 12. Какие данные нельзя удалять

- singleton-документы основных разделов;
- стабильные ключи карточек, этапов, вопросов, ответов и результатов;
- обязательные alt-тексты;
- реальные юридические данные;
- действующую ссылку на политику;
- `siteConfig.js`, пока миграция и fallback официально не завершены.
