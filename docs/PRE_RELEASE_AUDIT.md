# Speaker One — проверка перед публикацией

Дата проверки: 16 июля 2026 года.

## Что уже работает

- Production-сборка Vite проходит успешно.
- 24/24 теста контента Sanity проходят.
- 9/9 тестов серверного обработчика заявок проходят.
- Sanity Visual Editing и fallback на `siteConfig.js` предусмотрены архитектурой.
- Секреты Telegram не хранятся в публичном коде.
- Политика конфиденциальности содержит реальные реквизиты.

## Что блокирует публикацию

### 1. Не заполнены публичные параметры Sanity

Без них сборка использует резервный `siteConfig.js`, в котором остаются ссылки-заглушки.

Нужно добавить на хостинге:

```env
VITE_SANITY_PROJECT_ID=<реальный Project ID>
VITE_SANITY_DATASET=production
VITE_SANITY_API_VERSION=2026-07-15
VITE_SANITY_USE_CDN=true
```

### 2. Не указан реальный домен

Сейчас `robots.txt` и `sitemap.xml` генерируются с адресом `https://speaker-one.example/`.

До финальной сборки нужно указать реальный домен в опубликованном документе SEO / общих настройках Sanity.

### 3. В fallback-конфигурации остаются ссылки-заглушки

- Telegram-бот;
- Telegram-канал;
- VK;
- YouTube;
- Дзен.

Они должны быть заменены в Sanity и опубликованы. После этого production-сборка должна выполняться с заполненными `VITE_SANITY_*`.

### 4. Telegram-заявки требуют секретов хостинга

Нужно добавить только в защищённые настройки хостинга:

```env
TELEGRAM_BOT_TOKEN=<секретный токен>
TELEGRAM_CHAT_ID=<ID чата>
TELEGRAM_THREAD_ID=<необязательно>
LEAD_WEBHOOK_SECRET=<длинная случайная строка>
LEAD_ALLOWED_ORIGINS=https://ваш-домен.ru
LEAD_STATUS_ALLOWED_ORIGINS=https://адрес-studio.sanity.studio
```

В публичную сборку добавить:

```env
VITE_LEAD_ENDPOINT=https://ваш-домен.ru/api/leads
```

### 5. Нужно подтвердить публикацию изображений в Sanity

Изображения, оставшиеся в Draft, не попадут в обычный production-режим до нажатия `Publish`.

## Проверка после настройки

1. Изменить одно слово в Sanity и нажать `Publish`.
2. Выполнить `npm run build`.
3. Запустить `npm run preview`.
4. Проверить новый текст, все ссылки и изображения.
5. Отправить тестовую заявку и убедиться, что она пришла в Telegram.
6. Открыть `/privacy-policy.html`.
7. Проверить `robots.txt` и `sitemap.xml`: в них должен быть реальный домен.

## Итог

Кодовая база собирается и тесты проходят. Публикацию блокируют не программные ошибки, а отсутствующие реальные значения: домен, публичные ссылки, Sanity env и Telegram-секреты хостинга.
