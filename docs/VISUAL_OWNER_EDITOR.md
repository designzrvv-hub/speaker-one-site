# Визуальный редактор Speaker One

## Как изменить текст

1. Запустите сайт: `npm.cmd run dev:visual`.
2. В папке `studio` запустите Studio: `npm.cmd run dev`.
3. Откройте `http://localhost:3333` и войдите в Sanity.
4. Нажмите первый пункт **«РЕДАКТИРОВАТЬ САЙТ»**.
5. Нажмите на нужный текст в самом сайте и измените открывшееся поле справа.
6. Проверьте результат и нажмите **Publish**.

Пункт **«Все разделы»** — резервный способ работы. Для обычного изменения текста искать документ и поле вручную не нужно.

Предпросмотр обновляется во время редактирования автоматически через Sanity Live Content API. Ручная перезагрузка обычно не нужна. Публичный сайт при этом продолжает показывать только Published-версию.

## Как заменить фотографию

1. В «РЕДАКТИРОВАТЬ САЙТ» нажмите на фотографию.
2. Справа откроется используемое изображение.
3. Нажмите **Replace**, загрузите новый файл или перетащите его в поле.
4. Проверьте кадрирование и описание фотографии.
5. Нажмите **Publish**.

Перед первой публикацией перенесённого изображения подтвердите в его карточке, что у Вас есть право использовать файл на сайте. Этот флажок владелец отмечает лично; миграция не включает его автоматически.

Поддерживаются логотип, фото первого экрана, фото философии, фотографии эксперта и изображение Speech Lab. OG-изображение меняется в резервном разделе **SEO**, потому что оно не отображается на самой странице.

## Кнопки и Telegram

Нажмите на текст кнопки — справа откроется поле с её текущей подписью. Действие кнопки находится рядом в той же группе. Доступны переход к форме, Telegram, Telegram-бот, Telegram-канал, раздел сайта или безопасная внешняя ссылка.

Публичные Telegram-ссылки находятся в **«Все разделы» → «TELEGRAM И ЗАЯВКИ»**. Вставляйте только ссылку вида `https://t.me/...`. Bot Token и Chat ID в Sanity вводить нельзя: они хранятся только в защищённых переменных хостинга.

## Draft и Publish

- **Draft** — неопубликованное изменение. Оно видно в визуальном редакторе, но не обычным посетителям.
- **Publish** — опубликованная версия в Sanity.
- Публичный сайт использует только опубликованный контент. При текущей статической схеме после Publish нужна новая сборка и размещение сайта.
- Если Sanity недоступна или поле не прошло проверку, публичный сайт использует сохранённый `siteConfig.js`.

## Предпросмотр на телефоне

В верхней панели «РЕДАКТИРОВАТЬ САЙТ» переключите размер предпросмотра на телефон. Проверьте первый экран, длинные заголовки, карточки, Speech Lab, форму и подвал до публикации.

## Форма заявок

Тексты формы редактируются кликом по ним в сайте. Отправка заявок работает отдельно через `VITE_LEAD_ENDPOINT`. Сервер хранит `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` и при необходимости `TELEGRAM_THREAD_ID`. Эти значения никогда не помещаются в Studio или клиентский код.

## Если элемент не подсвечивается

1. Убедитесь, что открыли сайт через «РЕДАКТИРОВАТЬ САЙТ», а не отдельную публичную вкладку.
2. Проверьте, что документ существует и содержит текущее значение.
3. Обновите Studio один раз.
4. Если показано сообщение о недоступном preview, проверьте локальные server-only переменные по `docs/VISUAL_EDITING_ARCHITECTURE.md`.
5. Поля из резервного `siteConfig.js` намеренно не подсвечиваются: сначала их нужно перенести в Sanity.

## Что нельзя удалять

Не удаляйте singleton-документы, стабильные элементы массивов и описания фотографий. Не вводите в CMS пароли, токены ботов, Chat ID, write/read token, deploy token или другие секреты.
