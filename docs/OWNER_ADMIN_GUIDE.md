# Панель управления Speaker One

Эта инструкция описывает обычную работу владельца. Секреты Telegram и серверные настройки в Sanity не вводятся.

## Самые частые действия

### Изменить текст

1. Откройте `http://localhost:3333/` и войдите в Sanity.
2. В меню **2. ГЛАВНАЯ СТРАНИЦА** выберите нужный раздел.
3. Найдите поле по понятному названию и измените текст.
4. Исправьте красные ошибки и прочитайте жёлтые предупреждения.
5. Нажмите **Publish**.
6. Пересоберите и разместите сайт — автоматическая публикация пока не подключена.

### Заменить фотографию

1. Откройте **5. ИЗОБРАЖЕНИЯ** и нужное назначение фотографии.
2. Откройте существующий документ изображения.
3. Перетащите новый файл, настройте crop/hotspot.
4. Проверьте описание фотографии и право публикации.
5. Нажмите **Publish**.
6. Пересоберите сайт и проверьте desktop и mobile.

### Изменить кнопку

1. Откройте раздел, где находится кнопка.
2. Измените поле **Текст на кнопке**.
3. В поле **Что должна делать кнопка?** выберите действие.
4. Заполните только появившееся дополнительное поле.
5. Нажмите **Publish** и пересоберите сайт.

### Направить кнопку в Telegram-бота

1. Откройте **3. КНОПКИ И ССЫЛКИ → Социальные сети и Telegram**.
2. В поле **Telegram-бот** укажите полную ссылку `https://t.me/имя_бота`.
3. Включите **Показывать** и нажмите **Открыть и проверить ссылку**.
4. Опубликуйте документ.
5. Откройте нужную кнопку и выберите **Открыть Telegram-бота**.
6. Опубликуйте документ и пересоберите сайт.

## 1. Как запустить админку

```powershell
Set-Location -LiteralPath studio
npm.cmd run dev
```

Откройте [http://localhost:3333/](http://localhost:3333/). Если Studio просит войти, используйте личный аккаунт владельца проекта. Не передавайте общий пароль другим редакторам.

## 2. Где находится любой текст

Левое меню повторяет путь пользователя по сайту:

1. **НАЧАЛО** — реальные статусы заполнения и быстрые действия.
2. **ГЛАВНАЯ СТРАНИЦА** — первый экран, философия, компетенции, обо мне, опыт, этапы, Speech Lab, форма и подвал.
3. **КНОПКИ И ССЫЛКИ** — социальные сети, Telegram, навигация и основные кнопки.
4. **ЗАЯВКИ** — публичные тексты формы и безопасный статус endpoint.
5. **ИЗОБРАЖЕНИЯ** — медиа по назначению.
6. **SEO** — данные поиска и карточек ссылок.
7. **ЮРИДИЧЕСКАЯ ИНФОРМАЦИЯ** — реквизиты, Политика и согласие.
8. **НАСТРОЙКИ САЙТА** — название, домен, язык и публичные ссылки.

## 3. Как изменить слово или предложение

Откройте документ, измените поле и оставьте Draft, если хотите продолжить позже. Публикуйте только после проверки предупреждений. Не вставляйте HTML, JavaScript или служебные коды.

## 4. Как заменить фотографию

Используйте раздел **ИЗОБРАЖЕНИЯ**. Не вставляйте URL вручную. Для каждой фотографии заполните описание для поисковых систем, настройте область кадра и подтвердите право публикации. Не удаляйте старый media-документ, пока не проверили, где он используется.

## 5. Как изменить действие кнопки

Во всех подключённых CTA используется единое поле **Что должна делать кнопка?**:

- перейти к форме консультации;
- открыть основной Telegram;
- открыть Telegram-бота;
- открыть Telegram-канал;
- перейти к выбранному разделу;
- открыть безопасную внешнюю ссылку;
- скрыть кнопку.

Для перехода внутри сайта выбирайте раздел из списка. Не вводите `#contact` и другие технические якоря вручную.

## 6. Как изменить социальную сеть

Откройте **Социальные сети и Telegram**, выберите сеть, вставьте полную ссылку, включите отображение и используйте **Открыть и проверить ссылку**. Если ссылка помечена как заглушка, замените её перед релизом.

## 7. Как изменить вопрос Speech Lab

1. Откройте **Speech Lab → Вопросы**.
2. Выберите вопрос по его реальному тексту.
3. Измените вопрос или ответы.
4. Не меняйте группу **Служебные настройки результата** без необходимости.
5. У каждого вопроса оставьте минимум два ответа.
6. Проверьте полное прохождение диагностики перед Publish.

## 8. Как изменить текст формы

Откройте **Форма консультации**. Здесь меняются только публичные подписи, подсказки и сообщения. Endpoint, Bot Token, Chat ID и другие секреты здесь отсутствуют намеренно.

## 9. Как изменить Политику конфиденциальности

В CMS меняется только публичная ссылка на документ. Текущий путь — `/privacy-policy.html`. Сам HTML-файл обновляет разработчик. После изменения обязательно проверьте ссылку из формы и подвала. Юридический текст должен финально проверить владелец или юрист.

## 10. Как изменить SEO

Откройте **6. SEO**. Поля title и description показывают текущую длину; превышение рекомендаций создаёт предупреждение, но не блокирует сохранение. Укажите реальный основной домен и изображение 1200 × 630 px. SEO применяется при новой сборке, а не сразу после Publish.

## 11. Draft и Publish

- **Draft** — рабочая версия в Studio; посетители её не видят.
- **Publish** — опубликованная версия в Sanity.
- Публичный сайт получает Published-контент во время сборки. Поэтому после Publish нужна сборка и размещение нового `dist`.

## 12. Как проверить сайт перед публикацией

1. В корне выполните `npm.cmd run build`.
2. Выполните `npm.cmd run preview`.
3. Проверьте 320, 375, 768, 1024, 1440 и 1920 px.
4. Проверьте меню, все CTA, Speech Lab, форму, Footer и Политику.
5. Только затем разместите собранный сайт.

## 13. Визуальный редактор

Раздел **РЕДАКТОР САЙТА** остаётся дополнительным режимом. Для Draft-preview нужен защищённый preview-server и серверный read token. Если элемент не подсвечивается, используйте основное меню: оно остаётся полноценным способом редактирования.

## 14. Как подключить Telegram для заявок

Публичная ссылка на бота и доставка заявок — разные настройки. Ссылку владелец меняет в CMS. Доставка формы настраивается один раз на хостинге через серверную функцию. Подробности — в [TELEGRAM_OWNER_GUIDE.md](TELEGRAM_OWNER_GUIDE.md).

## 15. Где хранится Bot Token

Только в secret/environment settings серверной функции на хостинге под именем `TELEGRAM_BOT_TOKEN`. Не в Sanity, React, `siteConfig.js`, `.env.example`, переменных `VITE_*` или Git.

## 16. Где хранится Chat ID

Только в server environment под именем `TELEGRAM_CHAT_ID`. Он не отображается в Studio и не попадает в браузер.

## 17. Как отправить тестовую заявку

После размещения endpoint используйте защищённую команду из `docs/LEAD_ENDPOINT.md`, затем отправьте обычную тестовую форму с сайта. Успех подтверждён только если сообщение реально появилось в нужном Telegram-чате.

## 18. Если заявка не пришла

1. Проверьте статус endpoint в разделе **ЗАЯВКИ**.
2. Проверьте адрес `VITE_LEAD_ENDPOINT` и новую сборку сайта.
3. Проверьте Bot Token, Chat ID и права бота в secret settings хостинга.
4. Проверьте точный домен в `LEAD_ALLOWED_ORIGINS`.
5. Проверьте логи функции без публикации персональных данных.
6. Пока endpoint недоступен, форма сохраняет поля и показывает резервную Telegram-ссылку.

## 19. Как вернуть старый текст

Для неопубликованного Draft используйте **Discard changes**. Для опубликованного документа восстановите нужную версию из истории Sanity, если она доступна текущему тарифу, и снова Publish. `siteConfig.js` остаётся аварийным fallback.

## 20. Что нельзя удалять или публиковать

- Не удаляйте singleton-документы и стабильные элементы массивов без проверки.
- Не публикуйте Bot Token, Chat ID, read/write token, webhook secret, пароли и ключи CRM.
- Не меняйте технические веса Speech Lab без полного теста.
- Не публикуйте непроверенные реквизиты, факты, права на изображения и внешние ссылки.
- Не используйте `seed:overwrite` для обычного редактирования.

## Первичное заполнение текущим контентом

Seed уже подготовлен и содержит текущие тексты, карточки, Speech Lab, форму, Footer, SEO, реквизиты, ссылки, действия кнопок и изображения.

```powershell
Set-Location -LiteralPath studio
npm.cmd run seed:dry
npx sanity login
npm.cmd run seed:write
```

`seed:write` создаёт только отсутствующие Draft-документы и не перезаписывает изменения владельца. После записи откройте каждый документ, подтвердите права на изображения и Publish. Без входа Sanity CLI запись не выполняется.
