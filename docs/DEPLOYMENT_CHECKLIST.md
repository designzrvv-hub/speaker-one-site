# Speaker One — чек-лист публикации

## Домен и SEO

- [ ] Заменить временный `https://speaker-one.example` на реальный HTTPS-домен в `src/config/seoConfig.js`.
- [ ] Выполнить `npm.cmd run build`: сборка автоматически синхронизирует canonical, `og:url`, абсолютные URL изображений, `public/robots.txt`, `public/sitemap.xml` и `public/site.webmanifest`.
- [ ] Проверить итоговые canonical, `og:url`, `og:image`, title и description в собранном `dist/index.html`.
- [ ] Проверить, что выбранное изображение корректно кадрируется в Open Graph и Twitter Card; при необходимости подготовить отдельное превью формата около 1200 × 630 px.
- [ ] Открыть `/robots.txt` и убедиться, что указан реальный абсолютный URL sitemap.
- [ ] Открыть `/sitemap.xml`, проверить XML и URL главной страницы.
- [ ] Отправить sitemap в Яндекс Вебмастер и Google Search Console.

## Ссылки и юридические данные

- [ ] Заменить временные ссылки Telegram, ВКонтакте, YouTube и Дзен в `src/config/siteConfig.js`.
- [ ] Проверить ссылку на портфолио и доступ без авторизации.
- [ ] Проверить `/privacy-policy.html`, актуальность политики и соответствие фактическому способу обработки данных.
- [ ] Подтвердить перед публикацией все регалии и цифры в блоке эксперта.
- [ ] Сверить ИНН 744919295232, ОГРНИП 326745600003638 и имя предпринимателя с официальными документами.

## Форма и инфраструктура

- [ ] Настроить `VITE_LEAD_ENDPOINT` на собственную серверную функцию по инструкции `docs/LEAD_ENDPOINT.md`.
- [ ] Не направлять клиентский endpoint напрямую в Telegram Bot API и не добавлять токены в переменные `VITE_*`.
- [ ] Проверить успешную отправку, ошибки 4xx/5xx, timeout, offline и сохранение данных в форме.
- [ ] Проверить rate limiting, антиспам, серверную валидацию и безопасное логирование.
- [ ] Подключить SSL и убедиться, что весь сайт и endpoint работают только по HTTPS.

## Финальная проверка

- [ ] Выполнить `npm.cmd run build` и `git diff --check`.
- [ ] Проверить сайт на 320, 375, 768, 1024, 1440 и 1920 px без горизонтального скролла.
- [ ] Проверить клавиатурную навигацию, skip-link, focus-visible, labels и сообщения `aria-live`.
- [ ] Проверить Lighthouse для мобильной и десктопной версии.
- [ ] Проверить WebPageTest или аналогичный сервис на реальном домене.
- [ ] Проверить карточку ссылки в Telegram, ВКонтакте и других используемых мессенджерах.
- [ ] Проверить favicon, apple-touch-icon и manifest на реальном домене.
- [ ] Убедиться, что нет service worker и незапланированного offline-режима.
